###### 学んだこと
- relation = 実行前の検索条件(SQLへの命令セット)
- distinct = 重複なしでデータを取得する
- scope = モデルに記載する。検索条件に名前をつけて保存することで再利用できる。
###### つまずいたポイント・解決方法 
- **問題**:scope :by_category, ->(category_id) { where(category_id: category_id) }
category_idがたくさんあって混乱。
- **学び**:
  - ->(category_id)は引数。今回でいうとsearchメソッドでの、
  - `relation = relation.by_category(category_id) if category_id.present?`
  - この(category_id)の値が、スコープの引数の部分に入る。
  - { where(category_id: category_id) }に関してはArticleテーブルのcategory_idから探してねという意味。左のcategory_idに先ほどの引数の数値が入るという仕組み。
