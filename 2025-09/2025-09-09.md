## つまずいたポイント・解決方法
- **問題**:FATAL:  database "myapp_development" does not exist
- **解決策**:git reset, git clean, dockerの古いコンテナの削除、database.ymlの名前修正、rails db:createDB再作成
- **学び**:
rails newをする場合は古いdockerコンテナやvolume,DB等が残ったままになっていないか要注意。
- ①git status で確認してコミットorリセット
- ②docker compose down
- ③dockerの不要コンテナの削除 docker container prune
- ④volume(DBデータ)の削除 docker volume prune
- ⑤rm -rf ファイル名

- 再度rails newする場合のポイント
  - DB作成(docker compose run --rm web rails db:create)する前に、`database.yml`でアプリ名が正しいかdevelopmentを確認する
